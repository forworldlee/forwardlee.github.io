Storm并行计算的架构

nimbus是Storm集群中的主节点，也是一个supervisor节点，通过zookeeper分布式调度管理supervisor节点集群。nimbus分配supervisor和woker节点的数量进行并行计算。woker在指派一个Topology给executor和task执行。一个Topology是这样定义的：

Topology是一个逻辑上的概念，一个Topology包含一个Spout和若干Bolt，Spout主要负责从数据源加载数据和并行计算的初始化工作，Spout初始化一条待发送的数据（叫做Tuple）发送给Bolt进行处理，Tuple源源不断的从Spout发送给下游各个Bolt，形成了数据流Stream。最终由task处理Bolt中计算逻辑。由n个task同时执行Bolt代码，叫做并行度n。Bolt在处理完数据后，发送给下游Bolt时，需要对数据进行分组处理，如果数据随机发到下游Bolt，那么叫做随机负载均衡流分组Shuffle Grouping；如果数据是按照某一个或多个字段分组后，同一个分组的数据固定的被发送到同一个下游Bolt中执行，这叫做按字段分组Fields Grouping。还有其他不常用的分组方式，可忽略。

Topology代码

WordCountTopology代码，特别注意Spout、Bolt之间的关联关系。Bolt在执行流分组是要绑定上游节点的名称，例如：第一个SplitSentenceBolt.shuffleGrouping("RandomSentence");绑定的是spout的名称RandomSentence,第二个WordCountBolt.fieldsGrouping("SplitSentence", new Fields("word"));绑定的是上游SplitSentenceBolt的名字SplitSentence，如果名称绑定错误会抛出org.apache.storm.generated.InvalidTopologyException: null
